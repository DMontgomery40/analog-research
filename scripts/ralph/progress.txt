## 2026-02-13 - US-005
- Implemented accessibility and mobile optimization
- Added skip-to-content link as first focusable element (`<a href="#main-content">`) with `id="main-content"` on `<main>`
- Fixed color contrast for WCAG AA compliance:
  - Light mode accent changed from `#0d9488` (3.59:1) to `#0f766e` (5.24:1 on bg, 5.47:1 on white)
  - Dark mode: submit button and skip-link text changed to `#042f2e` on `#2dd4bf` (7.77:1)
  - Dark mode: logo-icon text changed to `#042f2e` for contrast on teal background
- Added `*:focus-visible` styles with 2px solid accent outline for all interactive elements
- Removed `outline: none` from form `:focus` (was hiding browser focus rings)
- Added `min-height: 44px` for touch targets on form inputs, submit button, available-now-link, and logo
- Added `aria-describedby="contact-help"` on form, with `id="contact-help"` on the help text paragraph
- Added `@media (max-width: 359px)` breakpoint for 320px viewport: reduced padding, smaller hero text
- All sections already had `aria-labelledby` from US-001
- All form inputs already had matching `label[for]` / `input[id]` pairs
- Files changed: index.html
- **Learnings for future iterations:**
  - WCAG AA requires 4.5:1 contrast for normal text, 3:1 for large text (18pt+ or 14pt+ bold)
  - `#0d9488` (Tailwind teal-600) fails WCAG AA on white backgrounds — `#0f766e` (Tailwind teal-700) passes
  - `focus-visible` is better than `focus` for keyboard accessibility — it doesn't trigger on mouse clicks
  - Decorative elements with `aria-hidden="true"` are exempt from contrast requirements, but still good to fix
  - The `available-now-link` uses `var(--fg)` bg with `var(--bg)` text which auto-inverts in dark mode — good pattern
